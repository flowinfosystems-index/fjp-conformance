The Judgment-Grounded Record

Every FJP response — whether you build your own or call the resolver — carries four components:

signalwhat changed, with sources
judgmentwhy it matters, referencing the signal
actionwhat should happen, referencing the judgment
falsifierwhat would make this wrong, checkable

The references form a chain. The falsifier makes the decision checkable after the fact. An agent that emits this record can be audited. One that doesn't, can't.

Three tools

Each call returns a complete Judgment-Grounded Record. One credit per call.

judge—Act on this, wait, or skip? With reasoning and a falsifier.
weigh—Decisive, material, or peripheral? Ranked with justification.
match—Does this fit that? With a relevance score and why.

Test your agent's output

FJP-CONF validates that your agent emits well-formed Judgment-Grounded Records. Stdlib-only Python, no dependencies. Use it whether you build your own judgment or call the resolver.

$ pip install fjp-conformance
$ fjp-conform record.json --level 2

Four conformance levels:

L0
StructuralWell-formed record, all four components present.
L1
GroundedSignal is attributable, reference chain resolves.
L2
FalsifiableFalsifier is concrete and checkable, not a catch-all.
L3
AccountableRecords retained, re-evaluable against what actually happened.
