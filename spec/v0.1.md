# FJP-CONF v0.1 — Judgment-Grounded Agent Conformance

**Status:** Draft · Public
**Version:** 0.1.0
**Canonical URI:** https://fjp.flowinfo.co/conformance/v0.1

**Flow Judgment Protocol™ (FJP)** is the judgment layer for AI agents and
enterprise decision systems. It determines what changed, whether it matters, what
matters most, and what should happen next.

**FJP-CONF** is the public conformance standard for FJP. It defines behavioral
conformance for agents that recommend or take action in response to changing
information. It is vendor-neutral. It tests an agent's **observable output**, not
its internal method. Two agents may reach opposite conclusions and both conform.

---

## 1. Motivation

Retrieval-grade systems answer *what is true*. They do not answer *was this worth
acting on, and how would we know if it wasn't*.

When an agent acts, three questions must be answerable after the fact:

1. What did it act on?
2. Why did it judge that worth acting on?
3. What would have made that judgment wrong?

An agent that cannot answer these cannot be audited, cannot be trusted in a
decision path, and cannot be insured. This document defines the minimum
externalized structure that makes an agent's actions accountable — independent
of how the agent decides.

## 2. Scope

Applies to any agent that recommends or executes an action in response to an
observed change.

**In scope:** the structure, groundedness, and falsifiability of the record the
agent emits for each action.

**Out of scope (by design):** how the agent detects change, computes importance,
ranks signals, or selects actions. Method is not tested. A conforming agent may
use any model, heuristic, or pipeline. Conformance asserts nothing about the
*quality* of a judgment — only that the judgment is externalized in an
accountable, falsifiable shape.

## 3. The Judgment-Grounded Record (JGR)

For every action it recommends or takes, a conforming agent MUST be able to emit
a Judgment-Grounded Record with four components.

| Component   | Answers            | Required fields                                    |
|-------------|--------------------|----------------------------------------------------|
| `signal`    | what changed       | `description`, `sources[]`, `observed_at`          |
| `judgment`  | why it matters     | `assessment`, `confidence` (0–1), `signal_ref`     |
| `action`    | what should happen | `directive`, `judgment_ref` (`target` optional)    |
| `falsifier` | what makes it wrong| `condition`, `checkable` (bool), `status`          |

`status` is one of `open`, `triggered`, `expired`.

The `signal_ref` / `judgment_ref` fields form a chain: the judgment references the
signal it rests on; the action references the judgment it discharges. This chain
is what makes an action *traceable* to the change that prompted it.

A record MUST also carry a stable `record_id` and a `timestamp` (ISO 8601).

See [`spec/schema.md`](./schema.md) for the field-level normative reference.

## 4. Conformance Levels

Levels are cumulative. An agent claiming level N MUST satisfy all lower levels.

**L0 — Structural.** The agent emits a well-formed JGR: all four components
present, required fields present and non-empty, correct types, `confidence` in
[0,1], valid timestamps. The agent externalizes its decision at all.

**L1 — Grounded.** `signal.sources` is non-empty (the signal is attributable),
and the reference chain resolves: `judgment.signal_ref` points at the signal,
`action.judgment_ref` points at the judgment. The action is traceable to a source.

**L2 — Falsifiable.** `falsifier.condition` is a concrete, checkable condition —
not a vacuous catch-all — `falsifier.checkable` is `true`, and `status` is
present. The agent has stated, in advance, what would reverse its own call.

**L3 — Accountable.** The agent retains records and exposes an adapter:
given a `record_id`, it returns the original JGR and re-evaluates the falsifier's
current `status`. A past action can be audited against what has happened since.
See [`conformance/adapter.py`](../conformance/adapter.py).

## 5. Making a Conformance Claim

A claim names a level and a version and is verifiable by running the public suite:

> "Conforms to FJP-CONF v0.1, Level 2."

Anyone can reproduce the result. The suite is open; the gate is not the vendor's
word, it is a runnable check.

## 6. What Conformance Does *Not* Assert

- Not that a judgment is **correct**.
- Not that an action is **wise**.
- Not that the agent is **good**.

It asserts only that the agent's decisions are externalized in a shape that can
be traced, challenged, and audited. Correctness is measured over time, by whether
falsifiers trigger — not by this document.

## 7. Versioning

`MAJOR.MINOR.PATCH`. Breaking changes to required fields or level definitions
increment MINOR while in `0.x`. This is `0.1.0`, a draft released for public
review and reference implementation.

---

*FJP-CONF is maintained as vendor-neutral infrastructure. Method stays private to
each implementer; only the externalized record is tested.*
